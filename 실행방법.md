# DownloadOrganizer 실행 방법

다운로드 폴더를 안전하게 분석·분류하고 보고서를 만드는 도구입니다.
**기본은 미리보기(Dry-run)** 이며, 명시적으로 확인하기 전에는 어떤 파일도 이동하지 않습니다.

---

## 0. 사전 준비

- Python **3.11 이상** 설치
- Windows PowerShell 기준 (다른 OS도 가능)

버전 확인:

```powershell
python --version
```

---

## 1. 설치

프로젝트 폴더(`DownloadOrganizer`)에서:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -e .
```

> 가상환경 활성화가 차단되면 한 번만:
> `Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass`

테스트까지 함께 설치하려면:

```powershell
pip install -e ".[dev]"
```

## 2. CLI 실행

실행은 **analyze → apply → undo** 로 분리됩니다.

### 2-1. 분석 (미리보기, 파일 이동 없음)

```powershell
download-organizer analyze --scan-root "C:\Users\사용자\Downloads"
```

- 파일을 읽기만 하고 **이동하지 않습니다.**
- 보고서(`download_organizer_report_*`, `bookmark_report_*`)가 `workspace\reports\` 에 생성됩니다.
- 출력 끝의 `hint` 에 **확인 토큰**과 그대로 실행할 apply 명령이 표시됩니다.

옵션:

| 옵션 | 설명 | 기본값 |
|------|------|--------|
| `--scan-root` | 정리할 다운로드 폴더 | 사용자 Downloads |
| `--output-root` | 보고서/이력 출력 폴더 | `.\workspace` |
| `--old-days` | 오래된 파일 기준(일) | `180` |
| `--dup-mode strict\|fast` | 중복 탐지 모드(strict=전체 해시, fast=앞 1MB) | `strict` |
| `--recursive` / `--no-recursive` | 하위폴더까지 스캔(기본 on). 정리 분류 폴더 자동 제외. 최상위만: `--no-recursive` | on |
| `--exclude-dir NAME` | 재귀 시 건너뛸 폴더명(반복 가능) | 없음 |
| `--ext-grouping` | 분류 폴더 아래 확장자 하위 폴더(예: `문서/pdf/`) | off |
| `--date-grouping none\|year\|month` | (확장자) 폴더 아래 수정일 기준 하위 폴더(예: `문서/pdf/2025/`) | `none` |
| `--include-category CAT` | 해당 분류만 이동(반복 가능) | 전체 |
| `--exclude-category CAT` | 해당 분류 제외(반복 가능) | 없음 |
| `--route-old` | 오래된 파일을 `_old_files`로 분리 | off |
| `--route-duplicates` | 중복 파일을 `_duplicates`로 분리(삭제 아님) | off |
| `--no-bookmarks` | 북마크 분석 제외(개인정보 보호) | 포함 |

> 부분 이동 시 `analyze`/`apply`에 **같은** include/exclude 옵션을 줘야 토큰이 일치합니다. 웹은 "이동 미리보기" 탭에서 파일별로 제외 선택.
| `--mask-bookmark-query` | 북마크 URL 쿼리/프래그먼트 제거 | off |
| `--exclude-domain DOMAIN` | 해당 도메인 북마크 제외(반복 가능) | 없음 |

### 2-2. 실제 이동 (계획 토큰 필요)

```powershell
download-organizer apply --scan-root "C:\Users\사용자\Downloads" --confirm-code <토큰>
```

- analyze가 출력한 **토큰을 `--confirm-code` 로 넘겨야만** 실행됩니다. 없거나 틀리면 100% 차단(`status: blocked`).
- 토큰은 *그 계획*에 묶여 있어, analyze 이후 폴더 내용이 바뀌면 토큰이 달라져 **오래된 계획이 실수로 적용되지 않습니다.**
- 이동 내역은 `workspace\history\move_history_YYYYMMDD_HHMMSS.json` 에 **한 건씩 즉시 기록**되어, 중간에 실패해도 되돌릴 수 있습니다.

> 옛 `run` 명령은 `analyze` 의 별칭으로만 남아 있고, `run --apply` 는 차단됩니다.

### 2-2-1. 설정 파일 (config.json)

분류 규칙·기준 일수·추가 차단 경로 등을 코드 수정 없이 바꿉니다.

```powershell
download-organizer init-config --path download_organizer.config.json
download-organizer analyze --config download_organizer.config.json
```

- `file_categories`, `bookmark_categories`는 있으면 전체 교체.
- `extra_blocked_roots`는 차단 경로 **추가만** (내장 차단은 제거 불가 — 안전).
- 명령행 옵션이 설정 파일보다 우선. 웹은 사이드바 "설정 파일 경로"에 입력.

### 2-3. 되돌리기 (Undo)

```powershell
# 먼저 무엇이 복원/건너뜀될지 미리보기 (이동 없음)
download-organizer undo --history-file "workspace\history\move_history_20260610_120000.json" --preview
# 실제 되돌리기
download-organizer undo --history-file "workspace\history\move_history_20260610_120000.json"
```

- 원래 위치에 같은 이름 파일이 있으면 **덮어쓰지 않고 건너뜁니다.**

### 2-4. 휴지통 정리 (선택)

중복 사본/오래된 파일/특정 분류를 **휴지통(복구 가능)** 으로 보냅니다. 영구 삭제는 하지 않습니다.

```powershell
# 미리보기(토큰 출력) — 휴지통 미접근
download-organizer clean --scan-root "C:\Users\사용자\Downloads" --trash-duplicates
# 실행 — 토큰 확인 필요
download-organizer clean --scan-root "C:\Users\사용자\Downloads" --trash-duplicates --confirm-code <토큰>
```

- `--trash-old`(오래된 파일), `--trash-category 문서`(분류 지정, 반복 가능) 조합 가능.
- 중복은 **그룹당 1개를 반드시 보존**합니다. 잘못 보내도 **Windows 휴지통에서 복원**할 수 있습니다.

## 3. 웹 UI (Streamlit) 실행

```powershell
streamlit run streamlit_app.py
```

브라우저가 열리면:

1. 왼쪽 **사이드바**에서 스캔/출력 경로, 오래된 파일 기준, 중복 모드(strict/fast), 북마크 포함·마스킹·제외 도메인 설정
2. **🔍 분석 (미리보기)** 클릭 → 요약 지표 + 탭(이동 미리보기·중복·오래된 파일·북마크) 확인
3. **📝 보고서 생성** 으로 MD/HTML/Excel 다운로드
4. 실제 이동은 **체크박스 + 계획 확인 토큰 입력** 2단계 후에만 활성화(화면에 표시된 토큰을 그대로 입력)
5. **↩️ 되돌리기** 탭: 이력 파일 선택 → **🔍 되돌리기 미리보기**로 복원/건너뜀 확인 후 **되돌리기 실행**
6. **🗑️ 휴지통 정리** 탭: 대상(중복/오래된/분류) 선택 → 미리보기(토큰) → 토큰 입력 후 휴지통으로 이동(복구 가능)

## 7. 안전 주의사항

- 이 도구는 **파일을 영구 삭제하지 않습니다.** 정리는 이동(`shutil.move`), 삭제는 `clean`의 **휴지통 이동(복구 가능)** 만 합니다.
- **선택한 다운로드 폴더 밖의 파일은 건드리지 않습니다.** (이동 직전 한 번 더 검증)
- 시스템/개인 경로는 차단됩니다: `C:\Windows`, `Program Files`, `ProgramData`, `AppData`, `Desktop`, `Documents`.
  - 따라서 스캔 경로로 바탕화면/문서 폴더 등은 지정할 수 없습니다(의도된 보호).
- 북마크 보고서에는 개인적인 사이트 정보가 포함될 수 있으니 공유 시 주의하세요. (`--no-bookmarks` 로 제외 가능)
